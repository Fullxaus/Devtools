**Базовые инструменты разработки**

JDK — Java Development Kit

Определение: Набор инструментов для разработки на Java, включающий компилятор (javac), библиотеки и JRE.

Контекст использования: Используется для разработки и запуска Java-приложений.

Пример: java -version — команда для проверки версии JDK.

Источник: https://docs.oracle.com/javase/8/docs/technotes/guides/

javac — Java Compiler

Определение: Компилятор Java, преобразующий исходный код (.java) в байт-код (.class).

Контекст использования: Используется для компиляции Java-кода.

Пример: javac Main.java — команда для компиляции Java-файла.

Источник: https://docs.oracle.com/javase/8/docs/technotes/tools/windows/javac.html

java — Java Runtime

Определение: Команда для запуска Java-приложений, являющаяся частью JRE.

Контекст использования: Используется для запуска Java-приложений.

Пример: java -jar myapp.jar — команда для запуска Java-приложения.

Источник: https://docs.oracle.com/javase/8/docs/technotes/guides/

PATH — System Variable

Определение: Системная переменная, содержащая пути к исполняемым файлам.

Контекст использования: Используется для запуска команд без указания полного пути.

Пример: echo $PATH — команда для просмотра значения переменной PATH.

Источник: https://www.gnu.org/software/bash/manual/html_node/Environment.html

JAVA_HOME — System Variable

Определение: Системная переменная, указывающая на директорию с установленной JDK.

Контекст использования: Используется другими инструментами для поиска Java.

Пример: echo $JAVA_HOME — команда для просмотра значения переменной JAVA_HOME.

Источник: https://docs.oracle.com/javase/8/docs/technotes/guides/

Определение: Система контроля версий — инструмент для отслеживания изменений в файлах проекта и координации работы нескольких разработчиков.

Контекст использования: Используется для управления версиями codebase.

Пример: Git, SVN, Mercurial — популярные VCS.

Источник: https://en.wikipedia.org/wiki/Version_control
Git — Distributed VCS

Определение: Распределённая система контроля версий, созданная Линусом Торвальдсом.

Контекст использования: Используется для отслеживания изменений в коде и сотрудничества между разработчиками.

**Непрерывная интеграция и доставка**

CI/CD — Continuous Integration/Continuous Delivery

Определение: Набор практик, автоматизирующих процессы сборки, тестирования и развертывания приложений.

Контекст использования: Используется для автоматизации процесса сборки, тестирования и развертывания программного обеспечения.

Пример: Использование Jenkins или GitLab CI/CD для автоматизации процесса сборки и развертывания.

Источник: https://en.wikipedia.org/wiki/CI/CD
GitHub Actions — GitHub Actions

Определение: Инструмент CI/CD, встроенный в GitHub, позволяющий автоматизировать различные задачи: сборку, тестирование, деплой и другие.

Контекст использования: Используется для автоматизации процесса сборки, тестирования и развертывания программного обеспечения на GitHub.

Пример: Создание workflow для автоматизации процесса сборки и тестирования.

Источник: https://docs.github.com/en/actions
Workflow — Workflow

Определение: Настраиваемый автоматизированный процесс в GitHub Actions, описываемый в YAML-файле (.github/workflows/).

Контекст использования: Используется для определения последовательности задач, которые необходимо выполнить.

Пример: Создание workflow для автоматизации процесса сборки и тестирования.

Источник: https://docs.github.com/en/actions/learn-github-actions/workflow-files
Runner — Runner

Определение: Сервер, на котором выполняются задачи CI/CD.

Контекст использования: Используется для выполнения задач CI/CD.

Пример: Использование GitHub Runner или самостоятельно настроенного сервера.

Источник: https://docs.github.com/en/actions/learn-github-actions/workflow-runners
Action — Action

Определение: Основной строительный блок в GitHub Actions, выполняющий определённую задачу.

Контекст использования: Используется для выполнения конкретных задач в workflow.

Пример: Использование встроенных или сторонних действий.

Источник: https://docs.github.com/en/actions/learn-github-actions/actions
Artifact — Artifact

Определение: Результат выполнения Workflow, который можно сохранить и использовать в дальнейшем.

Контекст использования: Используется для сохранения результатов выполнения workflow.

Пример: JAR-файл или отчёт о тестировании.

Источник: https://docs.github.com/en/actions/learn-github-actions/artifacts

**Тестирование и TDD**

JUnit — JUnit

Определение: Фреймворк для модульного (unit) тестирования в Java.

Контекст использования: Используется для написания и выполнения модульных тестов.

Пример: Написание тестов для классов и методов.

Источник: https://junit.org/

TDD — Test-Driven Development

Определение: Методология разработки ПО, при которой сначала пишутся тесты, а потом код, реализующий необходимую функциональность.

Контекст использования: Используется для обеспечения качества кода и сокращения количества ошибок.

Пример: Написание тестов перед написанием кода.

Источник: https://en.wikipedia.org/wiki/Test-driven_development
AssertJ — AssertJ

Определение: Библиотека для написания более читаемых и выразительных проверок (assertions) в тестах.

Контекст использования: Используется для написания тестов.

Пример: Написание assertions для тестов.

Источник: https://assertj.github.io/doc/
Unit Test — Unit Test

Определение: Тест, проверяющий небольшую, изолированную часть кода (обычно один метод или класс).

Контекст использования: Используется для проверки отдельных компонентов кода.

Пример: Написание тестов для методов и классов.

Источник: https://en.wikipedia.org/wiki/Unit_testing
Integration Test — Integration Test

Определение: Тест, проверяющий взаимодействие нескольких компонентов системы.

Контекст использования: Используется для проверки взаимодействия компонентов.

Пример: Написание тестов для взаимодействия компонентов.

Источник: https://en.wikipedia.org/wiki/Integration_testing
Test Coverage — Test Coverage

Определение: Мера того, какая часть кода покрыта тестами.

Контекст использования: Используется для оценки качества тестирования.

Пример: Измерение процента покрытия кода тестами.
